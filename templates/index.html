📖 nmap Reference & Scanning Tips

Essential Commands

nmap -sV -oX output.xml 192.168.1.0/24

Basic scan with version detection. The recommended starting point for most assessments.

nmap -A -oX output.xml 192.168.1.0/24

Aggressive scan: OS detection + version detection + script scanning + traceroute. More noise, more data.

nmap -sV --script vuln -oX output.xml [target]

Version detection + vulnerability scripts (NSE). Detects EternalBlue, Heartbleed, SMB issues, etc. Takes longer.

nmap -sV --script default,vuln -oX output.xml [target]

Default + vulnerability scripts combined. Best balance between coverage and speed.

Performance Flags — Large Networks

FlagDescription
--top-ports 100Only scan the 100 most common ports. Up to 10× faster on large subnets. Use 150–500 for better coverage.
-T4Aggressive timing. Faster but slightly less accurate. Ideal for internal networks with low latency.
-T5Insane timing. Use only on localhost or very stable LANs — misses results on slower networks.
--min-rate 1000Send at least 1000 packets/sec. Good for fast internal scans.
-nSkip DNS resolution. Speeds up scans significantly when hostnames aren't needed.
--openShow only open ports. Reduces output noise considerably.
-PnSkip host discovery (treat all hosts as up). Useful when ICMP is blocked.
--exclude [IPs]Exclude specific hosts (e.g. printers, fragile equipment).

Large Subnet Strategy

nmap -sV --top-ports 150 -T4 -n --open -oX fast_scan.xml 10.0.0.0/16

Fast wide-coverage scan for large networks (/16 or bigger). Scans top 150 ports, aggressive timing, no DNS.

nmap -sV --top-ports 500 -T4 -oX medium_scan.xml 192.168.0.0/24

More thorough scan for a /24. Good balance between coverage and time (~5–15 min).

nmap -sV -p- -T4 -oX full_scan.xml [target]

Full port scan (all 65535 ports). Only for individual hosts — very slow on subnets.

Useful NSE Scripts

Script / CategoryWhat it does
--script vulnKnown CVEs: EternalBlue, Heartbleed, SMB issues, Shellshock, etc.
--script smb-security-modeSMB signing status. Critical for relay attack detection.
--script smb2-security-modeSMBv2 signing enforcement check.
--script ftp-anonChecks if FTP allows anonymous login.
--script ssh-auth-methodsLists SSH authentication methods (password vs key).
--script rdp-enum-encryptionChecks if RDP uses NLA or weak encryption.
--script dns-zone-transferAttempts DNS zone transfer (AXFR).
--script http-default-accountsTests default credentials on web interfaces.

Multi-File Scanning Tips

nmap -sV -oX scan_192.xml 192.168.1.0/24
nmap -sV -oX scan_10.xml 10.0.0.0/24

Scan different subnets separately and drop both XML files into NmapViz — they will be merged automatically, deduplicating hosts and combining port data.

nmap --top-ports 100 -oX quick.xml 192.168.1.0/24
nmap -sV -p 22,80,443,445 -oX detailed.xml 192.168.1.0/24

Combine a quick sweep with a detailed service scan on specific ports. Merge both in NmapViz for the most complete picture.

Stealth & Detection Evasion

FlagDescription
-sSSYN scan (default). Half-open, less likely to be logged.
-T1 / -T2Paranoid/sneaky timing. Very slow but harder to detect.
--randomize-hostsScan hosts in random order. Avoids sequential pattern detection.
-D RND:5Decoy scan with 5 random IPs. Obscures the real source.
-fFragment packets. Bypasses some firewalls/IDS.

⚠️ Only use evasion techniques on networks you are authorised to test.
